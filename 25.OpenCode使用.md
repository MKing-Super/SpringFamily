# opencode

## 安装路线

### 1 下载node.js

https://nodejs.org/zh-cn/download

```bash
# 验证 Node.js 版本：
node -v # Should print "v24.14.1".

# 验证 npm 版本：
npm -v # Should print "11.11.0".
```

> 异常处理：
>
> ### 以管理员身份运行PowerShell并修改执行策略
>
> 右键点击“开始菜单” → 选择“Windows PowerShell(管理员)”
>
> 运行以下命令：
>
> ```powershell
> Set-ExecutionPolicy RemoteSigned -Scope CurrentUser
> ```
>
> 输入 `Y`确认
>
> 关闭窗口，重新打开普通PowerShell

### 2 安装 OpenCode CLI

在 PowerShell（建议管理员身份）中执行以下命令之一：

```
npm install -g opencode-ai
```

**验证**：安装后，在终端输入 `opencode --version`，能显示版本号即成功

打开、分析已有项目的文件

```
code .
```

打开终端右击 - 属性，勾选 “将ctrl+shift +c/v用作。。。”

### 3 管理员cmd

1. 按下键盘上的 **`Win + X`** 组合键（Win 是键盘左下角带 Windows 图标的键）。
2. 在弹出的菜单中，直接按键盘上的 **`A`** 键。
3. 此时会弹出一个“用户账户控制”窗口，点击 **“是”**。

**效果**：直接弹出一个标题栏带有 **“管理员:”** 前缀的 CMD 窗口

缺陷：容易闪退

### 4 会话

| 启动命令              | 会话状态     | 表现                             | 适用场景                        |
| --------------------- | ------------ | -------------------------------- | ------------------------------- |
| `opencode`            | **全新会话** | 像打开新网页，不记得上次聊过什么 | 开始一个完全独立的新任务        |
| `opencode --continue` | **恢复上次** | 自动加载上次的对话历史和上下文   | **推荐！** 日常开发，接着写代码 |

**结论**：如果你在同一个项目目录下直接输 `opencode`，它确实每次都像“失忆”一样重新开始。请改用 `opencode --continue`来保持连续性。



## 常用命令

​		OpenCode 的命令分为两类：**终端启动命令**（在 CMD 或 Terminal 中输入）和 **TUI 斜杠命令**（在 OpenCode 交互界面中输入，以 `/`开头，如 `/init`）。

### 1 终端启动命令

（在 CMD / PowerShell / WT 中运行）

这些命令用于启动应用或直接执行任务，**不要**加 `/`前缀。

| 命令               | 说明                        | 常用示例                             |
| ------------------ | --------------------------- | ------------------------------------ |
| `opencode`         | 启动 TUI 交互界面（最常用） | `opencode .`（打开当前目录）         |
| `opencode run`     | 单次运行，不进入交互        | `opencode run "写个排序函数"`        |
| `opencode web`     | 启动 Web 版界面             | `opencode web --port 3000`           |
| `opencode auth`    | 登录/登出服务               | `opencode auth login --provider zen` |
| `opencode agent`   | 管理代理（Agent）           | `opencode agent list`                |
| `opencode upgrade` | 升级 OpenCode               | `opencode upgrade`                   |

### 2 TUI 斜杠命令

（在 OpenCode 界面中输入）

进入 `opencode .`后的界面，输入 `/`会触发命令面板。你问的 `/init`就在这里面。

#### 1. 项目与配置

| 命令        | 作用                                                         | 快捷键 (Ctrl+X) |
| ----------- | ------------------------------------------------------------ | --------------- |
| **`/init`** | **初始化项目**：扫描代码，生成 `agents.md`，让 AI 理解项目结构 | `i`             |
| `/models`   | 查看/切换 AI 模型（如 big-pickle, Claude）                   | `m`             |
| `/connect`  | 配置新的 AI 提供商（OpenAI/Claude 等）                       | -               |
| `/agents`   | 切换工作模式（如 `plan`只出方案，`build`真改代码）           | -               |
| `/review`   | 审查未提交的 Git 代码更改                                    | -               |

#### 2. 会话与编辑

| 命令        | 作用                                   | 快捷键 (Ctrl+X) |
| ----------- | -------------------------------------- | --------------- |
| `/new`      | 开始新对话（清空当前）                 | `n`             |
| `/sessions` | 查看/恢复历史会话                      | `l`             |
| `/undo`     | **撤销**上一步 AI 操作（回滚文件修改） | `u`             |
| `/redo`     | 重做撤销的操作                         | `r`             |
| `/compact`  | 压缩对话历史，节省 Token               | `c`             |
| `/share`    | 生成会话分享链接                       | `s`             |
| `/export`   | 导出对话为 Markdown                    | `x`             |

#### 3. 视图与辅助

| 命令       | 作用                              | 快捷键 (Ctrl+X) |
| ---------- | --------------------------------- | --------------- |
| `/theme`   | 切换界面主题（深色/浅色）         | `t`             |
| `/details` | 显示/隐藏工具执行详情             | `d`             |
| `/editor`  | 用外部编辑器（如 VSCode）写提示词 | `e`             |
| `/help`    | 查看完整命令列表                  | `h`             |
| `/exit`    | 退出程序                          | `q`             |
