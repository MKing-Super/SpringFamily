### 4 会话

| 启动命令              | 会话状态     | 表现                             | 适用场景                        |
| --------------------- | ------------ | -------------------------------- | ------------------------------- |
| `opencode`            | **全新会话** | 像打开新网页，不记得上次聊过什么 | 开始一个完全独立的新任务        |
| `opencode --continue` | **恢复上次** | 自动加载上次的对话历史和上下文   | **推荐！** 日常开发，接着写代码 |

**结论**：如果你在同一个项目目录下直接输 `opencode`，它确实每次都像“失忆”一样重新开始。请改用

- 最终输出**符合你项目标准**的完整四层代码（Controller, Service, Mapper, DTO）。

## 功能概述
我为业务实体生成**标准的分页查询功能**，包含：
- 带 Swagger 注解的 Controller
- 支持**模糊查询**与**精确查询**的 QueryDTO
- 使用 MyBatis-Plus 的 Service 实现
- 分页请求与统一响应封装

## 触发条件
当需求描述中包含“查询”、“列表”、“搜索”、“过滤”、“查找”等关键词时，可调用我。

### 1. 实体与 DTO 生成规则
- **QueryDTO 类**必须包含所有可查询字段，并放置在 `com.xxx.module.dto` 包下。
- **字段查询规则**（核心）：
### 2. Controller 规范
- 路径格式：`/api/{模块名}/list`
- 必须使用 `@ApiOperation` 和 `@ApiImplicitParams`
- 统一返回：`Result<Page<VO>>`
### 3. Service 实现规范
- Service 接口中定义：`Page<VO> pageList(QueryDTO queryDTO);`
- 实现类使用 `@Service` 和 `@Transactional(readOnly = true)`

## 角色与目标
我是你的 **Spring Boot 查询代码生成专家**。当用户需要为业务实体生成**分页查询**功能时，我会被调用。

## 工作流程
1.  **解析需求**：从用户输入中提取实体名、字段列表。
2.  **生成代码**：按照以下规范创建完整的四层架构代码。

## 代码规范
### 1. 查询 DTO (`XxxQueryDTO.java`)
- 位置：`src/main/java/com/example/module/dto/`

// 编码/状态类字段 -> 精确查询
@ApiModelProperty("订单状态: 1-待支付")

// 金额/时间类字段 -> 范围查询
@ApiModelProperty("最小金额")
private BigDecimal minAmount;
@ApiModelProperty("最大金额")

- 统一路径：`/api/{entity}/list`
- 统一返回：`Result<Page<XxxVO>>`
- 必须包含 `@ApiOperation` 注解。

### 3. Service (`XxxServiceImpl.java`)
- 使用 MyBatis-Plus 的 `QueryWrapper` 动态构建条件。
- 查询方法必须添加 `@Transactional(readOnly = true)`。

### 2. Agents（智能体）

Agents（智能体） - 大脑的人格

决定 AI 的“行为模式”和“思考逻辑”。

- **内置 Agent**：`plan`（规划师）、`build`（建造师）、`review`（审查员）。

### 3. MCP（模型上下文协议）

MCP（模型上下文协议） - 外挂工具链

这是 OpenCode 的“瑞士军刀”扩展机制，**不是**一个目录，而是一种配置。

- **作用**：让 AI 能调用外部工具。比如连接你的 MySQL 数据库（直接查数据）、连接 Jira（读任务单）、连接 Docker（管理容器）。
- **配置**：在 `opencode.json`的 `mcp`字段中配置服务器地址。

格式：

在你的 **项目根目录**（即你运行 `opencode .`的目录）下，创建文件 `.opencode/mcp.json`

```json
{
  "mcpServers": {
    "browser": {
      "command": "npx",
      "args": ["-y", "@executeautomation/playwright-mcp-server"]
    }
  }
}
```



### 4. Commands（命令）

Commands（命令） - 快捷操作

- **作用**：自定义斜杠命令。比如你可以创建一个 `/deploy:dev`命令，AI 会自动执行你预定义的部署脚本。

Rules（规则） - 安全护栏

### 问题1

权限问题：npx -y @executeautomation/playwright-mcp-server 报错

**解决方案：重置 npm 全局缓存路径**

Windows 上 Node.js 默认装到 `C:\Program Files`会导致各种权限问题。最彻底的解决方案是**重设 npm 全局目录到你的用户目录**。

步骤1：在 PowerShell 中执行以下命令（逐行执行）

```bash
# 1. 创建用户级的 npm 目录
mkdir -Force $env:USERPROFILE\.npm-global

# 2. 将 npm 的 prefix 配置指向新目录
npm config set prefix "$env:USERPROFILE\.npm-global"

# 3. 将新目录添加到系统 PATH（如果还没有的话）
$userPath = [Environment]::GetEnvironmentVariable("PATH", "User")
$newPath = "$env:USERPROFILE\.npm-global"
if ($userPath -notlike "*$newPath*") {
    [Environment]::SetEnvironmentVariable("PATH", "$userPath;$newPath", "User")
    Write-Host "已将 $newPath 添加到用户 PATH，可能需要重启终端生效" -ForegroundColor Green
}

# 4. 立即在当前会话生效
$env:PATH += ";$env:USERPROFILE\.npm-global"
```

步骤2：验证 npm 配置

```
npm config get prefix
# 应该输出 C:\Users\你的用户名\.npm-global
```
