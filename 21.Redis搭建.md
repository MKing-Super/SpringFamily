# Redis搭建

## CentOS7

```bash
sudo yum update -y  # 更新所有系统包，-y表示自动确认

sudo yum install -y gcc make wget tar  # 安装编译工具和下载工具

# 安装 C/C++ 编译器
sudo yum install -y gcc gcc-c++ make

# 安装其他编译依赖
sudo yum install -y kernel-devel

# 安装 EPEL 仓库
sudo yum install -y epel-release

# 然后重新尝试安装 jemalloc
sudo yum install -y jemalloc jemalloc-devel

# 安装 tcl（包含在 tcl-devel 中）
sudo yum install -y tcl tcl-devel

# 安装 Python3
sudo yum install -y python3 python3-pip


cd /usr/local/src  # 切换到常用源码目录
sudo wget https://download.redis.io/redis-stable.tar.gz  # 下载Redis源码包

sudo tar -xzvf redis-stable.tar.gz  # 解压下载的压缩包，-x解压，-z解gzip，-v显示详情，-f指定文件
cd redis-stable  # 进入解压后的目录
make test SINGLE=1	# 单机运行测试
sudo make  # 编译Redis

sudo make install  # 安装Redis到系统路径（默认/usr/local/bin/）

```

> 若报错：
>
> 清理之前的编译文件(若报错)
>
> make distclean
>
> 停止所有 Redis 实例
>
> pkill redis-server 2>/dev/null || true
> redis-cli shutdown 2>/dev/null || true

```bash
sudo mkdir -p /etc/redis  # 创建配置目录，-p表示自动创建父目录
sudo mkdir -p /var/log/redis  # 创建日志目录
sudo mkdir -p /var/lib/redis  # 创建数据存储目录

sudo cp /usr/local/src/redis-stable/redis.conf /etc/redis/  # 将源码中的默认配置复制到/etc/redis目录

sudo vim /etc/redis/redis.conf  # 使用vim编辑配置文件（也可用nano或vi）
```

redis.conf（修改局部内容）

```bash
bind 0.0.0.0  # 允许所有IP连接，如需限制改为bind 127.0.0.1
daemonize yes  # 以后台守护进程方式运行
logfile "/var/log/redis/redis.log"  # 指定日志文件路径
dir /var/lib/redis  # 指定数据存储目录
```

建 systemd 服务文件

```bash
sudo vim /etc/systemd/system/redis.service  # 创建systemd服务配置文件
```

内容

```bash
[Unit]
Description=Redis In-Memory Data Store  # 服务描述
After=network.target  # 在网络服务启动后启动

[Service]
Type=forking  # 服务类型为forking（后台运行）
ExecStart=/usr/local/bin/redis-server /etc/redis/redis.conf  # 启动命令
ExecStop=/usr/local/bin/redis-cli shutdown  # 停止命令
Restart=always  # 总是自动重启
User=redis  # 运行用户
Group=redis  # 运行用户组

[Install]
WantedBy=multi-user.target  # 多用户模式启动时启用
```

> 中文注释可能导致报错
>
> ```bash
> sudo rm -f /etc/systemd/system/redis.service  # 删除存在问题的服务文件
> 
> sudo vim /etc/systemd/system/redis.service  # 创建新的服务文件
> ```
>
> 内容
>
> ```bash
> [Unit]
> Description=Redis In-Memory Data Store
> After=network.target
>

> Type=forking

创建 Redis 专用用户

```bash
sudo useradd -r redis  # 创建系统用户redis，-r表示创建系统账户
sudo chown -R redis:redis /var/lib/redis /var/log/redis  # 修改目录属主，-R递归修改
```

防火墙配置

```bash
sudo firewall-cmd --permanent --add-port=6379/tcp  # 永久开放6379端口，--permanent永久生效
sudo firewall-cmd --reload  # 重新加载防火墙配置使生效

sudo firewall-cmd --list-all  # 查看防火墙所有配置
```

启动和测试 Redis

```bash
sudo systemctl daemon-reload  # 重载systemd配置，使新服务生效

sudo systemctl start redis  # 启动redis服务

sudo systemctl enable redis  # 设置redis开机自动启动

sudo systemctl status redis  # 查看redis服务运行状态
```

测试连接

```bash
redis-cli ping  # 连接Redis并发送ping命令，应返回PONG表示正常

# 方法1：连接时指定密码
redis-cli -a 123456 ping  # -a指定密码连接

# 方法2：先连接再认证
redis-cli  # 进入redis命令行
auth yourpassword  # 认证密码
ping  # 测试连接

redis-cli set testkey "Hello Redis"  # 设置键值
redis-cli get testkey  # 获取键值，应返回"Hello Redis"
```

验证安装成功

```bash
redis-server --version  # 查看安装的Redis版本

ps aux | grep redis  # 查看redis进程是否运行

netstat -tlnp | grep 6379  # 查看6379端口是否被监听
# 或者使用：
ss -tlnp | grep 6379  # 新版本推荐使用ss命令

```

常用命令

```bash
# 查看实时日志
sudo tail -f /var/log/redis/redis.log  # -f持续查看新增内容

# 重启服务
sudo systemctl restart redis  # 重启redis服务

# 停止服务
sudo systemctl stop redis  # 停止redis服务

# 查看服务是否开机自启
sudo systemctl is-enabled redis  # 检查是否启用开机启动

# 进入交互模式执行多条命令
redis-cli  # 进入Redis命令行界面
# 在Redis命令行中可以执行：
# keys *  # 查看所有键（生产环境慎用）
# info  # 查看Redis信息
# exit  # 退出


# 查看错误日志
sudo journalctl -u redis.service -f  # 查看redis服务日志，-f实时查看

# 测试配置文件语法
redis-server /etc/redis/redis.conf --test-memory  # 测试配置

# 检查配置文件路径
sudo systemctl show redis --property=ExecStart  # 查看服务启动命令
```
